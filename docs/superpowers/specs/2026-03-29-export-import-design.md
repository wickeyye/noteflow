# NoteFlow 笔记导出和导入功能设计

## 概述

为 NoteFlow 添加笔记的导出和导入功能，支持 Markdown 格式的单个笔记导出、所有笔记批量导出为 ZIP、以及 Markdown 和 ZIP 文件的导入。

## 设计目标

1. **导出功能**：支持导出单个笔记和所有笔记
2. **导入功能**：支持导入单个 Markdown 文件和 ZIP 批量导入
3. **用户体验**：简单直观的操作流程，清晰的错误提示
4. **数据安全**：导入不覆盖现有笔记，避免数据丢失

## 用户需求

- 导出笔记用于备份和分享
- 批量导出所有笔记用于数据迁移
- 导入 Markdown 文件创建新笔记
- 批量导入备份的笔记

## 技术方案

### 技术选型

**方案 A：使用浏览器原生 API + 成熟库（选定）**

- 使用 `file-saver` 库处理文件下载
- 使用 `jszip` 库创建和解析 ZIP 文件
- 使用 HTML5 File API 处理文件上传

**优点**：
- 纯前端实现，无需后端
- 库成熟稳定，使用广泛
- 实现简单，代码量少
- 维护成本低

**依赖安装**：
```bash
npm install file-saver jszip
npm install --save-dev @types/file-saver
```

## 功能设计

### 1. 导出单个笔记

**触发条件**：
- 用户点击"导出当前笔记"
- 当前有选中的笔记

**文件命名**：
- 格式：`{笔记标题}.md`
- 特殊字符处理：替换为安全字符（`/\:*?"<>|` → `_`）
- 示例：`我的笔记.md`

**文件内容**：
- 纯 Markdown 格式
- 只包含笔记内容，不包含元数据

**错误处理**：
- 如果没有选中笔记，按钮禁用
- 导出失败时显示错误提示

### 2. 导出所有笔记

**触发条件**：
- 用户点击"导出所有笔记"
- 至少有一条笔记

**文件命名**：
- 格式：`NoteFlow-备份-{YYYY-MM-DD}.zip`
- 示例：`NoteFlow-备份-2026-03-29.zip`

**ZIP 内容**：
- 每个笔记一个 .md 文件
- 文件命名：`{笔记标题}.md`
- 重名处理：添加序号后缀（如 `笔记.md`, `笔记-2.md`）

**文件结构**：
```
NoteFlow-备份-2026-03-29.zip
├── 笔记1.md
├── 笔记2.md
└── 笔记3.md
```

**错误处理**：
- 如果没有任何笔记，按钮禁用
- ZIP 创建失败时显示错误提示

### 3. 导入 Markdown 文件

**触发条件**：
- 用户点击"导入"按钮
- 选择 .md 文件

**文件处理**：
- 文件名（去除 .md 扩展名）作为笔记标题
- 文件内容作为笔记内容
- 自动设置当前日期为更新时间
- 默认不收藏（isFavorite: false）

**标题处理**：
- 移除 .md 扩展名
- 如果标题为空，使用"未命名笔记"
- 不检查重名，允许创建同名笔记

**导入后操作**：
- 自动选中新创建的笔记
- 显示成功提示："已导入笔记：{标题}"

**错误处理**：
- 文件格式不支持：显示"仅支持 .md 和 .zip 文件"
- 文件读取失败：显示"文件读取失败，请重试"
- 文件过大（>10MB）：显示警告

### 4. 导入 ZIP 文件（批量导入）

**触发条件**：
- 用户点击"导入"按钮
- 选择 .zip 文件

**文件处理**：
- 解析 ZIP 文件，提取所有 .md 文件
- 忽略非 .md 文件（不报错）
- 每个 .md 文件创建一个新笔记

**导入逻辑**：
- 按文件名排序后依次导入
- 每个文件独立处理，一个失败不影响其他
- 导入的笔记添加到笔记列表顶部

**导入后操作**：
- 选中第一个导入的笔记
- 显示统计信息："成功导入 X 条笔记"

**错误处理**：
- ZIP 文件损坏：显示"ZIP 文件损坏，无法解析"
- 没有找到 .md 文件：显示"ZIP 中没有找到 Markdown 文件"
- 部分导入失败：显示"成功导入 X 条，失败 Y 条"

## 数据流设计

### 导出流程

```
用户点击导出按钮
  ↓
获取笔记数据（单个或全部）
  ↓
转换为 Markdown 格式
  ↓
[单个] 创建 Blob → 触发下载
[批量] 创建 ZIP → 添加文件 → 触发下载
  ↓
显示成功提示
```

### 导入流程

```
用户选择文件
  ↓
读取文件内容
  ↓
[.md] 解析 Markdown → 创建笔记
[.zip] 解压 ZIP → 提取 .md 文件 → 批量创建笔记
  ↓
添加到笔记列表
  ↓
保存到 LocalStorage
  ↓
选中新笔记
  ↓
显示成功提示
```

## 兼容性和限制

### 浏览器兼容性

- 支持现代浏览器（Chrome, Firefox, Safari, Edge）
- 需要支持 File API 和 Blob API
- 需要支持 async/await

### 文件大小限制

- 单个文件：建议不超过 10MB
- ZIP 文件：建议不超过 50MB
- 超过限制显示警告，但不阻止操作

### 文件名限制

- 最大长度：255 字符
- 特殊字符自动替换为下划线
- 空标题使用"未命名笔记"

## 风险和注意事项

**潜在风险**：
1. 大量笔记导出可能影响性能 - 通过异步处理和进度提示缓解
2. ZIP 文件解析可能消耗内存 - 限制文件大小并显示警告
3. 文件名冲突可能导致覆盖 - 使用序号后缀避免

**注意事项**：
- 导入总是创建新笔记，不覆盖现有笔记
- 文件名作为标题，用户可以在导入后修改
- 保持现有功能不变
- 确保数据持久化正常工作
