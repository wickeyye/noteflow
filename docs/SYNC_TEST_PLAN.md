# NoteFlow 云端同步测试计划

## 测试目标
验证 Supabase 云端同步功能正常工作，RLS 策略配置正确。

## 前置条件
1. 在 Supabase Dashboard 执行 `fix-rls-final.sql` 脚本
2. 确保 `notes` 表的 `user_id` 字段类型为 `uuid`
3. 确保 RLS 策略正确创建

## 测试步骤

### 阶段 1: RLS 策略验证

使用 `test-cloud-sync.html` 进行测试：

1. **打开测试页面**
   ```bash
   open test-cloud-sync.html
   # 或使用 Live Server
   ```

2. **登录账户**
   - 输入邮箱和密码
   - 点击"登录"
   - 确认连接状态显示"已连接"

3. **测试 CRUD 操作**
   - 点击"测试插入笔记" - 应成功插入
   - 点击"测试查询笔记" - 应显示刚插入的笔记
   - 点击"测试更新笔记" - 应成功更新
   - 点击"测试删除笔记" - 应成功删除

4. **检查表结构**
   - 点击"检查表结构"确认字段完整

### 阶段 2: 应用集成测试

1. **启动应用**
   ```bash
   npm run dev
   ```

2. **登录测试**
   - 使用相同账户登录
   - 观察控制台是否有同步日志

3. **笔记同步测试**
   - 创建新笔记
   - 等待 30 秒自动同步
   - 检查控制台是否有 "🔄 自动同步完成" 日志

4. **多设备同步测试**
   - 在浏览器 A 创建笔记
   - 在浏览器 B 登录同一账户
   - 确认笔记同步过来

### 阶段 3: 冲突解决测试

1. **离线编辑测试**
   - 断开网络
   - 编辑本地笔记
   - 恢复网络
   - 确认同步正常

2. **冲突解决测试**
   - 在设备 A 编辑笔记
   - 在设备 B 编辑同一笔记
   - 验证 Last Write Wins 策略

## 预期结果

- ✅ INSERT/SELECT/UPDATE/DELETE 都成功
- ✅ 自动同步每 30 秒执行一次
- ✅ 多设备间数据同步正常
- ✅ 冲突时保留最新版本

## 故障排查

### "new row violates row-level security policy"
- 检查 RLS 策略是否正确创建
- 检查 `user_id` 字段类型是否为 `uuid`
- 检查用户是否已认证

### "auth.uid() 返回 null"
- 确认用户已登录
- 检查 JWT token 是否过期

### 数据未同步
- 检查浏览器控制台日志
- 确认 `hasSyncedRef` 只在首次登录时触发
